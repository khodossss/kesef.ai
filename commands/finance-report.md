---
description: Финансовый отчёт из il-bank-live с дедупом банк/карта и сверкой баланса
---

Сделай финансовый отчёт по данным MCP-сервера `il-bank-live`.

Период: $ARGUMENTS (если пусто — спроси или возьми текущий + прошлый месяц).

Следуй стандартному флоу:

1. **Сначала спроси интерактивно (выбором, через AskUserQuestion — не решай сам),
   обновлять ли данные перед отчётом.** `refresh` интерактивный: открывает браузер,
   для Hapoalim пользователь вводит **SMS-код**, для Isracard проходит **Cloudflare**
   — то есть только когда пользователь за компьютером. Предложи: обновить оба /
   конкретный банк / считать из уже собранных (укажи, когда обновляли в последний раз).
2. Если обновляем — вызови `mcp__il-bank-live__refresh` по каждому провайдеру
   (по одному, интерактивно). Если нет — переходи к анализу из базы.
3. **Дедуп банк/карта:** траты = детальные покупки Isracard; погашения карт в банке
   (`מסטרקרד` / `כרטיסי אשראי`) исключай. Никогда не складывай их вместе.
4. **Покажи конкретную сверку баланса** через `mcp__il-bank-live__reconcile`
   (currentBalance, pendingCardBill, availableBalance, ledger) — цифрами, не на словах.
5. **Структура отчёта:** траты (по категориям/продавцам), доходы, повторяющиеся
   списания/подписки, баланс за период (доходы − расходы), сверка.

Данные бери через `get_transactions` / `list_accounts` / `reconcile`. Помни: баланс
двигают лумповые погашения карт, а не детальные покупки — это разные числа.
