---
description: Финансовый отчёт из il-bank-live с дедупом банк/карта и сверкой баланса
---

Сделай финансовый отчёт по данным MCP-сервера `il-bank-live`.

Период: $ARGUMENTS

Следуй стандартному флоу:

1. **Сначала определись с периодом — это первый вопрос, не про обновление.**
   Если период задан в запросе (`$ARGUMENTS`) — бери его. Если не задан —
   **спроси у пользователя, за какой период делать анализ** (через
   AskUserQuestion; удобный дефолт-вариант — текущий + прошлый месяц). Дальше не
   иди, пока период не определён.
2. **Проверь, покрыт ли этот период данными в БД** — `status` (когда каждый
   провайдер обновляли, сколько транзакций) и при необходимости `get_transactions`
   за период.
   - Если по нужным провайдерам данные за период **уже есть и достаточно свежие** —
     **НЕ спрашивай про обновление**, сразу переходи к анализу из базы. (Пример: просят
     «за прошлый месяц», а прошлый месяц уже лежит в БД — просто делаешь анализ.)
   - Если данных за период нет или они устарели (например, просят текущий месяц, а
     последнее обновление недельной давности) — **тогда** спроси интерактивно (выбором,
     через AskUserQuestion — не решай сам), обновлять ли: обновить оба / конкретный банк /
     считать из уже собранных. Укажи, когда обновляли в последний раз.
     `refresh` интерактивный: открывает браузер, для Hapoalim пользователь вводит
     **SMS-код**, для Isracard проходит **Cloudflare** — то есть только когда пользователь
     за компьютером.
3. Если обновляем — вызови `mcp__il-bank-live__refresh` по каждому провайдеру
   (по одному, интерактивно). Если нет — переходи к анализу из базы.
4. **Дедуп банк/карта:** траты = детальные покупки Isracard; погашения карт в банке
   (`מסטרקרד` / `כרטיסי אשראי`) исключай. Никогда не складывай их вместе.
5. **Покажи конкретную сверку баланса** через `mcp__il-bank-live__reconcile`
   (currentBalance, pendingCardBill, availableBalance, ledger) — цифрами, не на словах.
6. **Структура отчёта:** траты (по категориям/продавцам), доходы, повторяющиеся
   списания/подписки, баланс за период (доходы − расходы), сверка.

Данные бери через `get_transactions` / `list_accounts` / `reconcile`. Помни: баланс
двигают лумповые погашения карт, а не детальные покупки — это разные числа.
